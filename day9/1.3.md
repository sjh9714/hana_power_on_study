# Tokenization(토큰화)이란?

LLM에서 Tokenization은:

> “문장을 AI가 처리할 수 있는 작은 조각(Token)으로 분리하는 과정”

입니다.

LLM의 가장 첫 번째 단계이며 매우 중요합니다.

전체 흐름:

```text id="g4mqyo"
문장 입력
 ↓
토큰화(Tokenization)
 ↓
숫자(Token ID)
 ↓
벡터(Embedding)
 ↓
Transformer 처리
```

---

# 1. 왜 토큰화가 필요한가?

컴퓨터는 문장을 그대로 이해하지 못합니다.

예시:

```text id="9bslmf"
나는 오늘 학교에 갔다
```

컴퓨터 입장에서는 그냥 긴 문자열입니다.

그래서 먼저:

```text id="23a52v"
작은 단위로 분리
```

해야 합니다.

---

# 2. Token(토큰)이란?

토큰은:

```text id="jlwmcj"
LLM이 처리하는 최소 단위
```

입니다.

토큰은 반드시 “단어”일 필요는 없습니다.

토큰 종류:

| 형태      | 예시       |
| --------- | ---------- |
| 단어      | 학교       |
| 글자      | 학 / 교    |
| 부분 단어 | play / ing |
| 기호      | ! @ #      |
| 숫자      | 123        |
| 공백 포함 | " hello"   |

---

# 3. 가장 쉬운 예시

문장:

```text id="u2j6wo"
I love AI
```

토큰화:

```text id="fdn2w5"
[I] [love] [AI]
```

---

# 4. 실제 LLM은 더 복잡하다

예시:

```text id="v8w5h5"
unbelievable
```

↓

```text id="g6v7vw"
un + believ + able
```

처럼 분리될 수 있습니다.

---

# 5. 왜 부분 단어(Subword)를 사용하는가?

매우 중요합니다.

모든 단어를 저장하면:

```text id="l6ww2u"
사전 크기 폭발
```

이 발생합니다.

예시:

* running
* runner
* runners
* runnable

를 모두 저장하면 비효율적입니다.

그래서:

```text id="1v5vxm"
run + ning
run + ner
```

처럼 나눕니다.

---

# 6. OOV(Out Of Vocabulary) 문제 해결

예전 AI 문제:

```text id="x8p13i"
처음 보는 단어 처리 불가
```

예시:

```text id="r0mdk9"
ChatGPTUltraMegaX
```

새 단어라도:

```text id="m72v48"
Chat + GPT + Ultra + Mega + X
```

로 분리 가능.

# 9. 한국어 토큰화는 더 어렵다

영어는 띄어쓰기가 비교적 명확합니다.

하지만 한국어는 조사와 어미가 붙습니다.

예시:

```text id="6pw5wb"
학교에갔다
```

↓

가능한 분리:

```text id="abj6xk"
학교 / 에 / 갔다
```

또는:

```text id="obkljp"
학 / 교 / 에 / 갔 / 다
```

---

# 10. 한국어의 어려움

예시:

```text id="m4x13e"
먹었습니다
```

내부적으로:

```text id="h2wwn5"
먹 + 었 + 습니다
```

처럼 의미 단위가 섞여 있습니다.

# 11. 토큰화 방식 종류

대표 방식:

| 방식                      | 설명       |
| ----------------------- | -------- |
| Word Tokenization       | 단어 기준    |
| Character Tokenization  | 글자 기준    |
| Subword Tokenization    | 부분 단어 기준 |
| Byte-level Tokenization | 바이트 기준   |

---

# 12. Word Tokenization

예시:

```text id="4j8nlv"

```

↓

```text id="6k4zhv"

```

장점:

* 이해 쉬움

단점:

* 새로운 단어 처리 어려움

# 13. Character Tokenization

예시:

```text id="4y02di"

```

↓


단점:

# 15. BPE(Byte Pair Encoding)

가장 유명한 토큰화 방식 중 하나.

OpenAI GPT 계열에서 사용됩니다.

---

# 16. BPE 핵심 아이디어

초기:


```text id="wq8sg9"

```

더 자주 등장하면:

```text id="6vwjlwm"
low
```

로 합칩니다.

# 31. 실제 코드 예시 (Python)

OpenAI GPT 스타일 토큰화:

```python
import tiktoken

encoding = tiktoken.encoding_for_model("gpt-4o")

text = "안녕하세요. LLM을 공부하고 있습니다."

tokens = encoding.encode(text)

print("토큰 수:", len(tokens))
print(tokens)

```

Hugging Face tokenizer:

```python
# Hugging Face Transformers 라이브러리에서 AutoTokenizer 클래스를 가져온다.
#
# AutoTokenizer: 모델에 맞는 Tokenizer를 자동으로 선택하여 로드해준다.
#
# 예:
# bert-base-uncased -> BertTokenizer
# roberta-base      -> RobertaTokenizer
# gpt2              -> GPT2Tokenizer
from transformers import AutoTokenizer

# ============================================================
# 사전 학습된(pretrained) BERT 토크나이저 다운로드 및 로드
# ============================================================
#
# "bert-base-uncased"
#
# bert      : BERT 모델
# base      : 12개의 Transformer Layer 사용
# uncased   : 대소문자를 구분하지 않음
#
# 최초 실행 시:
#
# 1. Hugging Face Hub 접속
# 2. config.json 다운로드
# 3. tokenizer_config.json 다운로드
# 4. vocab.txt 다운로드
# 5. tokenizer.json 다운로드
#
# 다운로드된 파일은 C:\Users\<사용자>\.cache\huggingface\hub 에 저장된다.
# 두 번째 실행부터는 캐시된 파일을 사용하므로 다운로드 과정 없이 빠르게 실행된다.
#
tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased")


# ============================================================
# 문자열을 토큰(Token)으로 분리
# ============================================================
#
# Tokenizer는 문장을 작은 단위(Token)로 나눈다.
#
# 예)
#
# "I love AI"
#
# ->
#
# ['i', 'love', 'ai']
#
#
# BERT는 WordPiece 알고리즘을 사용한다.
#
# 사전에 존재하는 단어는 그대로 사용하고
# 사전에 없는 단어는 여러 조각으로 분리한다.
#
# 예)
#
# "playing"
#
# ->
#
# ['play', '##ing']
#
#
# ## 의미
#
# "앞 토큰 뒤에 붙는 부분"
#
# play + ing
#
# ->
#
# playing
##BERT 사전에 unbelievable 이 존재함 
tokens = tokenizer.tokenize("unbelievable")

# ============================================================
# 토큰 출력
# ============================================================
#
# 결과: ['unbelievable']
# 이유: BERT Vocabulary(사전)에 "unbelievable" 단어가 존재하기 때문
#
print(tokens)


#BERT 사전에 unbelievably 이 존재 하지 않음
tokens = tokenizer.tokenize("unbelievably")

print(tokens)
# 출력 결과 : ['un', '##bel', '##ie', '##va', '##bly']

```
