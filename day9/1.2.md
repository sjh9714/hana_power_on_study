# LLM(Large Language Model) 구조 이해하기

LLM은 사람의 언어를 이해하고 생성하는 거대한 AI 모델입니다.
대표적으로 OpenAI의 GPT, Google의 Gemini, Meta의 Llama 등이 있습니다.

쉽게 말하면:

> “엄청난 양의 문서를 읽고, 다음 단어를 예측하도록 학습된 초대형 신경망”

입니다.

---

# 1. 가장 쉬운 비유 — “문장 자동완성 엔진”

예를 들어 사람이 다음처럼 입력합니다.

```text
오늘 날씨가 매우
```

LLM은 다음 단어를 예측합니다.

```text
좋다
춥다
덥다
```

즉,

```text
이전 단어들을 보고
다음 단어를 확률적으로 예측
```

하는 구조입니다.

---

# 2. LLM 전체 구조

전체 흐름은 아래와 같습니다.

```text
입력 문장
   ↓
토큰(Token) 변환
   ↓
임베딩(Embedding)
   ↓
Transformer 블록 여러 층
   ↓
다음 단어 확률 계산
   ↓
출력 생성
```

---

# 3. 실제 예시로 이해하기

사용자 입력:

```text
나는 오늘 학교에 갔다
```

LLM 내부에서는 다음처럼 처리됩니다.

---

# 4. Step 1

## 4.1 Tokenization (토큰화)

문장을 작은 단위로 분리합니다.

예시:

```text
나는 / 오늘 / 학교에 / 갔다
```

또는 실제로는:

```text
[1532, 9911, 881, 4421]
```

같은 숫자로 변환됩니다.

이 숫자를 Token ID 라고 합니다.

---

## 4.2 왜 숫자로 바꾸는가?

컴퓨터는 문자열 자체를 이해하지 못합니다.

따라서:

```text
"학교"
```

를

```text
881
```

같은 숫자로 바꾸어 계산합니다.

---

# 5. Step 2

# 5.1 Embedding (임베딩)

토큰 숫자를 벡터(Vector)로 변환합니다.

예시:

```text
881 → [0.12, -0.44, 0.91, ...]
```

즉:

```text
단어 → 숫자 벡터
```

로 변환합니다.

---

## 5.2 임베딩이 중요한 이유

의미가 비슷한 단어는 가까운 위치에 배치됩니다.

예시:

```text
고양이 ≈ 강아지
자동차 ≠ 바나나
```

벡터 공간에서:

```text
고양이 ---- 강아지

자동차 ---------------- 바나나
```

처럼 거리가 달라집니다.

---

# 6. 핵심 구조 — Transformer

LLM의 핵심은 Transformer 구조입니다.

2017년 Google 논문:

```text
Attention Is All You Need
```

에서 등장했습니다.

---

# 9. Transformer 핵심 아이디어

Transformer는:

```text
문장 전체를 동시에 바라봄
```

기존 RNN처럼 한 단어씩 처리하지 않습니다.

예시:

```text
나는 사과를 먹었다.
```

여기서:

```text
먹었다
```

가:

```text
나는
사과를
```

모두를 동시에 참고합니다.

---

# 10. 가장 중요한 개념 — Attention

Attention은:

```text
어떤 단어를 얼마나 중요하게 볼 것인가
```

를 계산합니다.

---

# 11. Attention 예시

문장:

```text
철수는 배가 아파 병원에 갔다.
```

여기서:

```text
갔다
```

를 이해할 때 모델은:

```text
철수
배
아파
병원
```

중 특히:

```text
병원
아파
```

에 높은 집중(attention)을 줍니다.

---

# 12. Self-Attention 구조

Self-Attention은:

```text
문장 내부 단어끼리 서로 관계를 계산
```

합니다.

즉:

```text
각 단어가
다른 모든 단어를 참고
```

합니다.

---

# 13. Attention 내부 계산

Attention은 다음 3개를 사용합니다.

```text
Q = Query
K = Key
V = Value
```

---

# 14. 비유로 이해하기

도서관 비유:

| 구성  | 의미                |
| ----- | ------------------- |
| Query | 내가 찾고 싶은 정보 |
| Key   | 책 제목             |
| Value | 실제 책 내용        |

---

# 15. 실제 계산 흐름

Attention 점수:

$$
Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

의미:

```text
Q와 K의 유사도 계산
→ 중요도 확률 계산
→ V를 가중합
```

입니다.

---

# 16. Multi-Head Attention

LLM은 여러 시각으로 문장을 봅니다.

예시:

| Head  | 보는 관점 |
| ----- | --------- |
| Head1 | 문법      |
| Head2 | 의미      |
| Head3 | 시간 관계 |
| Head4 | 감정      |

즉 여러 Attention을 동시에 수행합니다.

---

# 17. Transformer 블록 구조

하나의 블록은 보통:

```text
입력
 ↓
Self-Attention
 ↓
Feed Forward Network
 ↓
정규화(Norm)
 ↓
출력
```

구조입니다.

그리고 이것이 수십~수백 층 반복됩니다.

---

# 18. Feed Forward Network (FFN)

Attention 후에:

```text
비선형 계산
```

을 수행하는 작은 신경망입니다.

예시:

```text
Linear
↓
ReLU/GELU
↓
Linear
```

---

# 19. Positional Encoding

Transformer는 단어 순서를 모릅니다.

따라서 위치 정보를 추가합니다.

예시:

```text
나는 학교에 갔다
학교에 나는 갔다
```

단어는 같아도 의미가 다릅니다.

이를 위해 위치 벡터를 추가합니다.

---

# 20. 학습 과정

LLM 학습 핵심:

```text
다음 단어 맞추기
```

입니다.

예시:

```text
나는 밥을 ___
```

정답:

```text
먹었다
```

모델은 확률을 계산합니다.

---

# 21. 확률 예시

```text
먹었다  85%
샀다     5%
잤다     3%
```

이런 식으로 계산합니다.

---

# 22. 손실 함수(Loss)

예측이 틀리면 오차를 계산합니다.

대표적으로:

```text
Cross Entropy Loss
```

를 사용합니다.

---

# 23. 역전파(Backpropagation)

오차를 기반으로:

```text
가중치 수정
```

을 반복합니다.

즉:

```text
틀린 부분을 조금씩 수정
```

하면서 학습합니다.

---

# 24. GPU가 중요한 이유

LLM은 엄청난 행렬 연산을 수행합니다.

예시:

```text
수천억 개 파라미터
```

를 계산해야 합니다.

그래서:

* NVIDIA GPU
* TPU
* 대규모 클러스터

가 필요합니다.

---

# 25. 파라미터(Parameter)란?

파라미터는:

```text
모델의 기억 값
```

입니다.

예시:

* 7B → 70억 개
* 70B → 700억 개

---

# 26. 왜 모델이 커질수록 좋아지는가?

큰 모델은:

* 더 많은 패턴 학습
* 긴 문맥 이해
* 복잡한 추론 가능

해집니다.

---

# 27. LLM의 실제 생성 과정

예시 질문:

```text
한국의 수도는?
```

모델 내부:

```text
한국 → 수도 → 서울
```

확률 계산 후:

```text
서울
```

출력.

---

# 28. 토큰 단위 생성

LLM은 문장을 한 번에 생성하지 않습니다.

실제로는:

```text
서울
은
한국
의
수도
입니다
```

처럼 한 토큰씩 생성합니다.

---

# 29. Temperature

출력 다양성을 조절합니다.

* 낮음 → 안정적
* 높음 → 창의적

예시:

```text
0.1 → 거의 정답 위주
1.2 → 다양한 문장 생성
```

---

# 30. Top-K / Top-P

확률이 높은 후보만 선택합니다.

예시:

```text
상위 10개 단어만 후보
```
