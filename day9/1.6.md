# Feed Forward Network(FFN)란?

Transformer에서 FFN(Feed Forward Network)은:

> “Attention으로 모은 정보를 추가로 변환하고 학습하는 작은 신경망”

입니다.

Transformer 내부에서 매우 중요한 역할을 합니다.

전체 구조에서 보면:

```text id="2u1mbf"
입력
 ↓
Self-Attention
 ↓
FFN
 ↓
출력
```

형태입니다.

---

# 1. 왜 FFN이 필요한가?

많은 사람들이 처음에는:

```text id="zkm0hx"
Attention만 있으면 되는 것 아닌가?
```

라고 생각합니다.

하지만 Attention은:

```text id="oq90do"
단어 간 관계를 찾는 역할
```

만 잘합니다.

즉:

* 어떤 단어가 중요한지
* 어떤 단어와 관련 있는지

를 계산합니다.

---

# 2. Attention의 한계

Attention은 주로:

```text id="n29vzw"
관계 분석기
```

에 가깝습니다.

하지만:

* 의미 변환
* 특징 추출
* 비선형 패턴 학습

은 부족합니다.

이 역할을 FFN이 수행합니다.

---

# 3. 쉬운 비유

Transformer를 회사로 비유하면:

| 구성      | 역할           |
| --------- | -------------- |
| Attention | 회의           |
| FFN       | 실제 업무 처리 |

즉:

```text id="fxkg9f"
Attention이 정보를 수집
FFN이 실제 계산/가공
```

합니다.

---

# 4. 아주 쉬운 예시

문장:

```text id="bbm8e5"
나는 사과를 먹었다
```

Attention은:

```text id="m2m7j8"
먹었다 ↔ 사과
```

관계를 강하게 연결합니다.

하지만 FFN은:

```text id="l8r3yx"
"먹는 행동"
"음식 관계"
```

같은 복잡한 의미를 더 깊게 변환합니다.

---

# 5. FFN의 기본 구조

보통:

```text id="52e5t6"
Linear
↓
Activation
↓
Linear
```

구조입니다.

---

# 6. 수식으로 표현

Transformer 논문 기준:

$$
FFN(x)=\max(0,xW_1+b_1)W_2+b_2
$$

| 구성     | 의미           |
| -------- | -------------- |
| x        | 입력 벡터      |
| W1       | 첫 번째 가중치 |
| b1       | 편향           |
| max(0,x) | ReLU 활성화    |
| W2       | 두 번째 가중치 |

---

# 7. Linear Layer란?

Linear는:

```text id="rqdz1n"
행렬 곱셈
```

입니다.

예시:

$$
y=xW+b
$$

---

# 9. 의미

입력 벡터를:

```text id="7r1evi"
다른 특징 공간으로 변환
```

합니다.

---

# 10. 왜 두 번 Linear를 사용하는가?

첫 번째 Linear:

```text id="k0ww6n"
차원 확장
```

두 번째 Linear:

```text id="jlwmff"
원래 차원 복원
```

입니다.

---

# 11. 실제 차원 예시

GPT 계열 예시:

| 단계     | 차원  |
| -------- | ----- |
| 입력     | 4096  |
| FFN 내부 | 16384 |
| 출력     | 4096  |

---

# 12. 왜 차원을 크게 늘리는가?

더 복잡한 특징 학습 가능.

즉: 고차원 공간에서 복잡한 패턴 계산 합니다.

---

# 13. Activation Function

FFN 핵심 요소.

대표:

* ReLU
* GELU

---

# 14. 왜 Activation이 필요한가?

Activation 없으면 단순 선형 계산만 가능 합니다.
Activation은 비선형성 추가 역할 수행합니다.

---

# 15. 비선형성이 왜 중요한가?

언어는 매우 복잡합니다.

예시:

```text id="jlwmjj"
"은행"
```

은:

* 금융기관
* 강둑

두 의미 가능.

복잡한 의미 분리를 위해 비선형 필요.

---

# 16. ReLU

대표 함수:

$$
ReLU(x)=\max(0,x)
$$

---

# 17. ReLU 의미

| 입력 | 출력 |
| :--: | ---: |
| -3   | 0    |
| 2    | 2    |

즉:

```text id="jlwmkk"
음수 제거
```

---

# 18. GELU

최신 LLM에서 더 많이 사용.

GPT 계열도 사용.

---

# 19. GELU 특징

ReLU보다 부드럽게 동작.

$$
GELU(x)=x\Phi(x)
$$

---

# 20. 왜 GELU를 사용하는가?

언어 데이터는 부드러운 패턴 많음.

GELU는:

* 학습 안정성
* 성능 향상

에 유리.

---

# 21. FFN의 실제 역할

FFN은: Attention 결과를 더 복잡하게 가공 합니다.

---

# 22. Attention vs FFN

| 구성      | 역할      |
| --------- | --------- |
| Attention | 관계 분석 |
| FFN       | 특징 변환 |

---
