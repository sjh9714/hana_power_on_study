# 1. 생성형 AI(Generative AI)란?

생성형 AI는 데이터를 학습하여 새로운 콘텐츠를 생성하는 인공지능 기술이다.
기존 AI는 주로 “분석”, “분류”, “예측”을 수행했지만 생성형 AI는 “창작” 기능을 수행한다.
예를 들어 기존 AI는 고양이 사진을 인식하는 역할을 했다면 생성형 AI는 새로운 고양이 이미지를 직접 생성할 수 있다.
생성 가능한 데이터는 텍스트, 이미지, 음성, 영상, 코드 등 매우 다양하다.
대표적인 생성형 AI 서비스로는 OpenAI 의 ChatGPT가 있다.
생성형 AI는 대규모 데이터를 학습하면서 패턴과 확률을 이해한다.
그리고 입력된 문맥(Context)을 기반으로 가장 적절한 결과를 생성한다.
최근 생성형 AI는 업무 자동화와 생산성 향상의 핵심 기술로 평가받고 있다.
기업들은 고객 상담, 문서 작성, 개발 자동화 등에 적극 활용하고 있다.
또한 생성형 AI는 클라우드와 GPU 기술 발전으로 빠르게 성장하였다.
현재는 멀티모달 AI로 발전하면서 이미지·음성·텍스트를 동시에 처리할 수 있게 되었다.

### 예시

```text id="v2jfr4"
입력:
"우주복 입은 고양이 그림 만들어줘"

출력:
새로운 AI 이미지 생성
```

### 핵심 정리

* 생성형 AI는 새로운 데이터를 생성한다.
* 기존 AI는 분석/분류 중심이었다.
* 텍스트·이미지·음성·영상 생성이 가능하다.
* GPU와 딥러닝 발전이 핵심 배경이다.
* 현재 산업 전반에서 활용되고 있다.

# 2. 기존 AI와 생성형 AI의 차이

기존 AI는 입력 데이터를 분석하여 특정 결과를 예측하거나 분류하는 데 초점이 있었다.
예를 들어 이메일 스팸 판별, 얼굴 인식, 금융 이상 거래 탐지 등이 있다.
이러한 시스템은 정답(Label)이 존재하는 데이터를 기반으로 학습한다.
반면 생성형 AI는 새로운 콘텐츠를 만들어낸다는 점이 가장 큰 차이점이다.
생성형 AI는 단순 숫자 결과가 아니라 사람이 읽을 수 있는 자연스러운 결과를 제공한다.
또한 긴 문맥을 이해하고 연속적인 대화를 유지할 수 있다.
기존 AI는 특정 목적에 최적화된 좁은 기능 중심이었지만 생성형 AI는 범용성이 높다.
예를 들어 하나의 LLM이 번역, 요약, 코딩, 문서 작성까지 모두 수행할 수 있다.
최근 기업들은 기존 AI 시스템 위에 생성형 AI 인터페이스를 결합하고 있다.
이를 통해 사용자는 자연어로 시스템을 제어할 수 있게 되었다.
생성형 AI는 인간과 컴퓨터 간 인터페이스 자체를 변화시키고 있다.

### 예시 비교

| 구분 | 기존 AI        | 생성형 AI        |
| ---- | -------------  | ---------------- |
| 목적 | 분류/예측      | 콘텐츠 생성      |
| 결과 | 숫자/클래스    | 자연어/이미지    |
| 예시 | 사기 거래 판별 | 투자 리포트 작성 |
| 특징 | 특정 기능      | 범용 기능        |

### 핵심 정리

* 기존 AI는 예측 중심이다.
* 생성형 AI는 콘텐츠 생성 중심이다.
* 생성형 AI는 문맥 이해 능력이 뛰어나다.
* 하나의 모델이 여러 업무를 수행할 수 있다.
* 기업 시스템 인터페이스를 변화시키고 있다.

# 3. 생성형 AI의 핵심 원리

생성형 AI의 핵심 원리는 “다음 값을 예측”하는 것이다.
텍스트 생성에서는 다음 단어를 예측하고 이미지 생성에서는 다음 픽셀을 예측한다.
이 과정은 확률 계산 기반으로 수행된다.
예를 들어 “오늘 날씨가”라는 문장이 들어오면 AI는 다음 단어 후보를 계산한다.
이 계산은 수십억 개 이상의 파라미터(Parameter)를 기반으로 수행된다.
파라미터는 AI가 학습한 지식과 패턴을 저장하는 내부 숫자이다.
생성형 AI는 인터넷 문서, 코드, 책, 이미지 등을 대규모로 학습한다.
학습 과정에서 단순 암기가 아니라 관계와 패턴을 이해한다.
Transformer 구조는 긴 문맥을 동시에 고려할 수 있도록 지원한다.
따라서 생성형 AI는 사람처럼 자연스러운 결과를 생성할 수 있다.
결국 생성형 AI는 거대한 확률 기반 패턴 생성 시스템이라고 볼 수 있다.

### 예시

```text id="zwl3ps"
입력:
"대한민국의 수도는"

AI 내부 예측:
서울 (95%)
부산 (2%)
인천 (1%)
기타 (2%)
```

### 핵심 정리

* 생성형 AI는 “다음 값 예측” 기반이다.
* 확률 계산을 통해 결과를 생성한다.
* 파라미터는 학습된 지식을 저장한다.
* 대규모 데이터 학습이 핵심이다.
* Transformer가 문맥 이해를 지원한다.

# 4. 생성형 AI의 주요 종류

생성형 AI는 생성 대상에 따라 다양한 종류로 구분된다.
가장 대표적인 것은 텍스트 생성 AI이다.
텍스트 생성 AI는 질문 응답, 문서 작성, 번역, 코드 생성 등을 수행한다.
이미지 생성 AI는 텍스트 설명을 기반으로 새로운 이미지를 생성한다.
대표적인 이미지 생성 모델은 Stable Diffusion과 Midjourney이다.
음성 생성 AI는 사람 목소리를 합성하거나 특정 인물의 음성을 복제할 수 있다.
영상 생성 AI는 텍스트 기반으로 새로운 영상을 생성한다.
코드 생성 AI는 함수 자동 생성과 버그 수정 등을 수행한다.
최근에는 텍스트·이미지·음성을 동시에 처리하는 멀티모달 AI가 빠르게 발전하고 있다.
앞으로는 로봇·자율주행·디지털 휴먼과도 결합될 가능성이 높다.
생성형 AI는 거의 모든 데이터 형태를 생성할 수 있는 방향으로 발전 중이다.

# 5. 딥러닝과 생성형 AI

생성형 AI의 기반 기술은 딥러닝(Deep Learning)이다.
딥러닝은 인간의 신경망 구조를 모방한 인공신경망 기술이다.
입력층, 은닉층, 출력층 구조를 기반으로 데이터를 처리한다.
은닉층이 깊어질수록 더 복잡한 패턴을 학습할 수 있다.
딥러닝은 데이터에서 특징(Feature)을 자동으로 추출할 수 있다는 장점이 있다.
기존 머신러닝은 사람이 특징을 설계해야 했지만 딥러닝은 이를 자동화하였다.
생성형 AI는 수십억 개 이상의 파라미터를 학습하기 때문에 GPU가 필수적이다.
특히 NVIDIA GPU가 생성형 AI 학습에 많이 사용된다.
클라우드 환경에서는 GPU 클러스터를 사용하여 대규모 학습을 수행한다.
최근 생성형 AI 발전은 GPU 성능 향상과 함께 이루어지고 있다.
딥러닝은 현재 생성형 AI의 가장 중요한 기반 기술이다.

### 예시

```python id="8l7zq4"
# 간단한 딥러닝 예시
입력값 -> 은닉층 -> 출력값
"고양이 사진" -> 특징 추출 -> "고양이"
```

### 핵심 정리

* 생성형 AI의 핵심 기반은 딥러닝이다.
* 인공신경망 구조를 사용한다.
* GPU가 대규모 학습에 필수적이다.
* 특징 추출을 자동화한다.
* 데이터와 연산량이 많을수록 성능이 향상된다.

# 6. Transformer 구조

Transformer는 현재 생성형 AI의 핵심 구조이다.
2017년 발표된 “Attention Is All You Need” 논문에서 처음 제안되었다.
이전 RNN 구조는 긴 문장 처리 시 성능 저하 문제가 있었다.
Transformer는 병렬 처리 구조를 사용하여 속도를 크게 향상시켰다.
Attention 메커니즘을 이용해 중요한 단어 관계를 파악할 수 있다.
예를 들어 “철수가 사과를 먹었다”에서 누가 무엇을 했는지 이해할 수 있다.
Transformer는 Encoder와 Decoder 구조로 이루어진다.
GPT는 Decoder 중심 구조를 사용한다.
BERT는 Encoder 기반 구조를 사용한다.
현재 대부분의 LLM은 Transformer 기반이다.
Transformer는 생성형 AI 발전의 가장 중요한 혁신 기술로 평가된다.

### 예시

```text id="z5s4al"
입력:
"나는 어제 학교에서 친구를 만났다"

Attention:
"나" ↔ "만났다"
"친구" ↔ "학교"
관계를 학습
```

### 핵심 정리

* Transformer는 현재 LLM의 핵심 구조이다.
* Attention 메커니즘을 사용한다.
* 병렬 처리로 성능이 향상되었다.
* GPT와 BERT 모두 Transformer 기반이다.
* 긴 문맥 처리에 강점이 있다.

# 7. 토큰(Token) 개념

생성형 AI는 문장을 그대로 처리하지 않고 작은 단위로 나누어 처리한다.
이 단위를 토큰(Token)이라고 한다.
토큰은 단어, 글자 조각, 기호 등이 될 수 있다.
영어는 단어 기반으로 분리되는 경우가 많다.
한국어는 형태소 기반으로 분리되는 경우가 많다.
LLM은 입력과 출력을 모두 토큰 단위로 처리한다.
모델마다 한 번에 처리 가능한 최대 토큰 수가 존재한다.
최근 모델은 수십만 토큰까지 지원하고 있다.
토큰 수는 AI API 사용 비용 계산에도 사용된다.
따라서 실무에서는 토큰 최적화가 중요하다.
긴 문서를 처리하기 위해 문맥 관리 기술도 발전하고 있다.

### 예시

```text id="5m11rz"
문장:
"안녕하세요 반갑습니다"

토큰:
[안녕] [하세요] [반갑] [습니다]
```

### 핵심 정리

* 토큰은 AI가 처리하는 최소 단위이다.
* 입력과 출력 모두 토큰 기반이다.
* 모델마다 최대 토큰 제한이 있다.
* 토큰 수는 비용 계산 기준이다.
* 문맥 관리가 매우 중요하다.

# 8. LLM(Large Language Model)

LLM은 대규모 언어 모델을 의미한다.
텍스트 데이터를 대규모로 학습하여 자연어를 이해하고 생성한다.
대표 모델로 GPT, Llama, Gemini, Claude 등이 있다.
LLM은 수십억 개 이상의 파라미터를 학습한다.
파라미터 수가 많을수록 복잡한 패턴 학습이 가능하다.
LLM은 번역, 요약, 질문 응답, 코딩 등을 수행할 수 있다.
특히 자연스러운 대화 능력이 뛰어나다.
최근에는 멀티모달 기능이 추가되고 있다.
기업들은 고객 상담 AI와 업무 자동화에 LLM을 사용하고 있다.
오픈소스 LLM이 증가하면서 자체 구축도 활발해지고 있다.
LLM은 현재 생성형 AI 산업의 핵심 기술이다.

### 예시

```text id="xzy6lm"
질문:
"Python으로 리스트 정렬 방법 알려줘"

LLM 응답:
sorted(), sort() 함수 설명 및 예제 제공
```

### 핵심 정리

* LLM은 대규모 언어 모델이다.
* 자연어 생성과 이해가 가능하다.
* GPT, Llama 등이 대표 모델이다.
* 다양한 업무 자동화에 활용된다.
* 멀티모달 방향으로 발전 중이다.

# 9. 생성형 AI 학습 과정

생성형 AI는 먼저 대규모 데이터를 수집하는 단계부터 시작한다.
인터넷 문서, 책, 뉴스, 코드, 이미지 등이 학습 데이터로 사용된다.
다음 단계는 사전학습(Pretraining)이다.
이 과정에서 모델은 다음 단어 예측을 반복하면서 패턴을 학습한다.
사전학습에는 수천 개의 GPU가 사용될 수 있다.
이후 특정 목적에 맞게 파인튜닝(Fine-Tuning)을 수행한다.
예를 들어 금융 AI, 의료 AI, 법률 AI 형태로 추가 학습할 수 있다.
최근에는 RLHF 기술이 많이 사용된다.
이는 사람 피드백을 기반으로 모델을 개선하는 방식이다.
또한 최신 AI는 검색 기반 학습(RAG)과 결합되기도 한다.
생성형 AI는 여러 단계의 복잡한 학습 과정을 거쳐 완성된다.

### 예시

```text id="v8nq2e"
1단계: 데이터 수집
2단계: 사전학습
3단계: 파인튜닝
4단계: RLHF
5단계: 서비스 배포
```

### 핵심 정리

* 데이터 수집이 시작 단계이다.
* 사전학습으로 언어 패턴을 학습한다.
* 파인튜닝으로 특정 분야에 최적화한다.
* RLHF로 품질을 개선한다.
* GPU 클러스터가 필수적이다.

# 10. 생성형 AI 모델 종류

생성형 AI에는 다양한 모델 구조가 존재한다.
GPT 계열은 자연어 생성에 강점이 있다.
Diffusion Model은 이미지 생성에 많이 사용된다.
GAN은 생성자와 판별자가 경쟁하며 학습하는 구조이다.
VAE는 데이터 압축과 생성 기능을 함께 수행한다.
최근에는 멀티모달 모델이 증가하고 있다.
텍스트와 이미지 생성 기술이 통합되는 방향으로 발전 중이다.
모델마다 장단점과 활용 분야가 다르다.
예를 들어 GPT는 대화에 강하고 Diffusion은 이미지 품질이 뛰어나다.
기업들은 목적에 맞는 모델 구조를 선택하여 사용한다.
생성형 AI 모델은 계속 새로운 구조로 발전하고 있다.

### 예시

| 모델      | 특징           |
| --------- | -------------  |
| GPT       | 자연어 생성    |
| Diffusion | 이미지 생성    |
| GAN       | 경쟁 기반 생성 |
| VAE       | 압축 + 생성    |

### 핵심 정리

* 생성형 AI에는 다양한 모델 구조가 있다.
* GPT는 자연어 생성에 강하다.
* Diffusion은 이미지 생성 핵심 기술이다.
* GAN은 경쟁 구조를 사용한다.
* 멀티모달 모델이 증가 중이다.

# 11. 생성형 AI 활용 분야

생성형 AI는 거의 모든 산업에서 활용되고 있다.
교육 분야에서는 AI 튜터와 자동 문제 생성이 사용된다.
의료 분야에서는 진단 보조와 의료 문서 요약이 활용된다.
금융 분야에서는 이상 거래 분석과 투자 리포트 생성이 사용된다.
개발 분야에서는 코드 생성과 자동 테스트가 활발하다.
디자인 분야에서는 광고 이미지와 UI 시안 생성이 가능하다.
영상 제작에서는 자막 생성과 영상 편집 자동화가 발전 중이다.
클라우드 분야에서는 운영 자동화와 로그 분석에 활용된다.
Kubernetes 환경에서는 YAML 자동 생성도 가능하다.
최근에는 AI 에이전트 형태로 업무 자동화 수준이 높아지고 있다.
생성형 AI는 앞으로 대부분의 업무 환경에 통합될 가능성이 높다.

### 예시

```text id="0x0h0q"
DevOps 활용:
"쿠버네티스 배포 YAML 생성해줘"

AI:
Deployment + Service YAML 자동 생성
```

### 핵심 정리

* 생성형 AI는 산업 전반에서 사용된다.
* 개발 자동화 활용이 증가 중이다.
* 금융·의료·교육 분야 활용이 활발하다.
* DevOps 자동화에도 사용된다.
* AI 에이전트 형태로 발전 중이다.

# 12. 생성형 AI의 한계

생성형 AI는 강력하지만 여러 한계도 존재한다.
대표적인 문제는 Hallucination(환각) 현상이다.
이는 존재하지 않는 정보를 사실처럼 생성하는 문제이다.
또한 최신 정보를 즉시 반영하지 못할 수 있다.
학습 데이터의 편향(Bias)이 결과에 영향을 줄 수 있다.
대규모 GPU와 전력이 필요하기 때문에 비용이 매우 높다.
저작권과 개인정보 문제도 중요한 이슈이다.
프롬프트 공격(Prompt Injection) 같은 보안 문제도 존재한다.
생성형 AI 결과는 항상 검증이 필요하다.
기업 환경에서는 AI 거버넌스와 정책 관리가 중요하다.
따라서 인간의 통제와 검토가 반드시 필요하다.

### 예시

```text id="e4olcc"
질문:
"존재하지 않는 논문 알려줘"

AI:
가짜 논문 제목 생성 가능
```

### 핵심 정리

* 환각(Hallucination) 문제가 존재한다.
* 최신 정보 부족 문제가 있다.
* 편향과 보안 문제가 존재한다.
* 운영 비용이 매우 높다.
* 인간 검증이 반드시 필요하다.

## 2. RAG (Retrieval-Augmented Generation)

### 2.1 역할

RAG는 LLM이 학습된 데이터만 사용하는 것이 아니라, 외부 문서나 데이터베이스를 검색하여 답변에 활용하는 기술입니다. 사용자의 질문이 들어오면 먼저 관련 문서를 검색한 뒤, 검색된 내용을 Prompt에 추가하여 LLM이 답변을 생성합니다. 이 방식은 최신 정보나 회사 내부 문서처럼 모델이 학습하지 못한 데이터도 사용할 수 있게 해줍니다.
특히 PDF, Wiki, Notion, DB, 웹사이트 기반 챗봇 구축에 많이 사용됩니다.
환각(Hallucination)을 줄이고 답변 정확도를 크게 높일 수 있습니다.
핵심 기술로는 Embedding, Vector DB, Semantic Search가 사용됩니다.
대표 프레임워크는 LangChain, LlamaIndex 등이 있다.
실무에서는 금융 문서 검색, 사내 지식 검색, 고객센터 챗봇 등에 많이 적용된다.

### 2.2 구조

```text
사용자 질문
   ↓
Vector Search
   ↓
관련 문서 검색
   ↓
LLM Prompt에 삽입
   ↓
정확한 답변 생성
```

### 2.3 해결하는 문제

| 해결 가능    |
| ------------ |
| 최신 정보    |
| 사내 문서    |
| 환각 감소    |
| 긴 문서 검색 |

### 2.4 사용 기술

* LangChain
* LangGraph
* LlamaIndex
* FAISS
* ChromaDB
* Milvus
* Pinecone

### 2.5 예시

사용자:

> "하나은행 투자설명서에서 중도상환 수수료 알려줘"

동작:

```text
PDF 검색
→ 관련 문단 추출
→ GPT에게 전달
→ 정확한 답변
```

## 3. Fine-Tuning (파인튜닝)

### 3.1 역할

Fine-Tuning은 이미 학습된 LLM을 특정 목적에 맞게 추가 학습시키는 기술입니다.
예를 들어 금융, 의료, 법률 같은 전문 분야 데이터를 학습시켜 해당 분야에 특화된 모델을 만들 수 있습니다.
기업의 답변 스타일이나 업무 프로세스를 모델에 반영하는 데도 사용됩니다.
기존 모델 전체를 재학습하는 방식과 일부 파라미터만 학습하는 LoRA 방식 등이 있습니다.
전문 용어 이해 능력과 응답 품질을 높일 수 있다는 장점이 있습니다.
하지만 GPU 비용과 학습 데이터 품질이 매우 중요합니다.
최근에는 QLoRA 같은 저비용 학습 방식이 많이 사용됩니다.
실무에서는 고객 상담 AI, 금융 분석 AI, 의료 진단 AI 등에 활용됩니다.

### 3.2 예시

| 분야     | 목적             |
| -------- | ---------------- |
| 금융     | 금융 용어 학습   |
| 의료     | 의료 문맥 이해   |
| 법률     | 판례 스타일      |
| 고객센터 | 회사 답변 스타일 |

### 3.3 종류

| 방식             | 설명               |
| ---------------- | ------------------ |
| Full Fine-tuning | 전체 모델 재학습   |
| LoRA             | 일부만 학습        |
| QLoRA            | 저메모리 학습      |
| PEFT             | 파라미터 효율 학습 |

### 3.4 해결

* 전문성 부족
* 회사 말투
* 특정 업무 최적화

---

## 4. AI Agent

### 4.1 역할

AI Agent는 단순히 질문에 답변하는 것을 넘어 스스로 작업을 계획하고 실행하는 AI 시스템입니다.
LLM이 여러 단계의 작업을 분석하고 필요한 도구를 선택하여 문제를 해결합니다.
예를 들어 AWS 비용 분석, 이메일 발송, 데이터 수집 등을 자동 수행할 수 있습니다.
Agent는 Tool Calling, Memory, Workflow와 함께 동작하는 경우가 많습니다.
기존 챗봇보다 훨씬 자율적이며 업무 자동화 수준이 높습니다.
특히 반복 업무나 DevOps 자동화에 강력합니다.
대표 프레임워크는 LangGraph, CrewAI, AutoGen 등이 있습니다.
최근 AI 산업에서 가장 빠르게 성장하는 핵심 분야 중 하나입니다.

### 4.2 기존 LLM

```text
질문 → 답변
```

### 4.3 Agent

```text
질문
 ↓
계획
 ↓
도구 선택
 ↓
API 호출
 ↓
결과 분석
 ↓
최종 응답
```

### 4.4 예시

```text
"오늘 AWS 비용 분석해서 이메일 보내줘"
```

Agent:

1. AWS API 호출
2. 비용 분석
3. 그래프 생성
4. 이메일 전송

### 4.5 대표 프레임워크

* LangGraph
* CrewAI
* AutoGen
* Semantic Kernel

## 5. Tool Calling (Function Calling)

### 5.1 역할

Tool Calling은 LLM이 외부 함수나 API를 직접 호출할 수 있게 하는 기술입니다.
기존 LLM은 계산이나 실시간 데이터 처리에 약한 문제가 있었습니다.
이를 해결하기 위해 계산기, 검색 엔진, DB, API 등을 연결합니다.
예를 들어 날씨 조회, 환율 계산, Kubernetes 상태 조회 등을 자동 실행할 수 있습니다.
LLM은 어떤 도구가 필요한지 판단하고 적절한 함수를 호출합니다.
OpenAI Function Calling도 대표적인 예시입니다.
AI Agent의 핵심 구성 요소로 사용됩니다.
실무에서는 ERP, Slack, GitHub, 클라우드 API 연동 등에 활용됩니다.

### 5.2 예시

```python
get_weather(city="Seoul")
```

LLM:

```text
서울 날씨 API 호출 필요
```

→ 함수 자동 실행

### 5.3 해결 문제

| 문제             | 해결             |
| ---------------- | ---------------- |
| 계산 오류        | 계산기 사용      |
| 실시간 정보 부족 | API 호출         |
| 실행 불가        | 외부 시스템 연결 |

## 6. Vector Database

### 6.1 역할

Vector DB는 문장을 숫자 벡터로 변환하여 의미 기반 검색을 수행하는 데이터베이스입니다.
기존 DB는 키워드 기반 검색이 중심이었지만 Vector DB는 문맥 자체를 이해합니다.
예를 들어 “환불 정책”을 검색하면 “결제 취소 규정”도 찾아낼 수 있습니다.
LLM 기반 RAG 시스템의 핵심 저장소 역할을 합니다.
Embedding 모델을 사용하여 텍스트를 벡터로 변환한 후 저장합니다.
대표 제품으로는 Pinecone, Milvus, Chroma 등이 있습니다.
대규모 문서 검색과 AI 검색 엔진 구축에 매우 중요합니다.
최근 AI 시스템 대부분은 Vector DB를 함께 사용합니다.

### 6.2 기존 DB

```text
키워드 검색
```

### 6.3 Vector DB

```text
의미 검색
```

### 6.4 예시

질문:

> "환불 정책"

검색:

```text
"결제 취소 규정"
```

도 찾아냄

### 6.5 대표 기술

* Pinecone
* Milvus
* Chroma
* Weaviate
* FAISS

## 7. Memory (메모리 시스템)

### 7.1 역할

Memory는 AI가 이전 대화나 사용자 정보를 기억할 수 있게 하는 기술입니다.
기존 LLM은 세션이 끝나면 정보를 잃어버리는 문제가 있었습니다.
Memory를 사용하면 사용자의 관심사나 이전 작업을 유지할 수 있습니다.
예를 들어 “저는 Kubernetes를 공부 중입니다”라고 말하면 이후 대화에 반영됩니다.
단기 기억과 장기 기억 구조로 나누어 구현하는 경우가 많습니다.
AI Agent에서는 작업 상태를 유지하는 데도 사용됩니다.
개인화된 AI 서비스 구축에 매우 중요한 요소입니다.
실무에서는 AI 비서, 고객 지원 시스템 등에 활용됩니다.

### 7.2 종류

| 유형            | 설명        |
| --------------- | ----------  |
| Short-term      | 현재 대화   |
| Long-term       | 사용자 정보 |
| Semantic Memory | 지식 기억   |
| Episodic Memory | 행동 기록   |

### 7.3 예시

```text
"나는 Kubernetes 공부중"
```

다음 대화에서도 기억

## 8. Guardrails (안전장치)

### 8.1 역할

Guardrails는 LLM의 위험한 응답을 제어하기 위한 안전장치 기술입니다.
환각(Hallucination), 욕설, 개인정보 유출, 정책 위반 등을 방지합니다.
사용자의 악의적인 Prompt Injection 공격도 차단할 수 있습니다.
입력 검증과 출력 검증을 모두 수행하는 경우가 많습니다.
기업 환경에서는 보안과 규제 대응 때문에 매우 중요합니다.
NVIDIA NeMo Guardrails 같은 프레임워크가 대표적입니다.
최근에는 AI 보안(AI Security) 분야와 함께 빠르게 발전 중입니다.
금융·의료·공공기관 AI 서비스에서는 필수 요소로 여겨집니다.

### 8.2 방어

| 문제            | 예시             |
| --------------- | ---------------- |
| 개인정보 유출   | 주민번호         |
| 욕설            | toxic filtering  |
| jailbreak       | 우회 프롬프트    |
| 환각            | 검증             |
| 정책 위반       | 보안 차단        |

### 8.3 기술

* NVIDIA NeMo Guardrails
* Guardrails AI
* OpenAI Moderation

## 9. MCP (Model Context Protocol)

### 9.1 역할

MCP는 LLM과 외부 시스템을 표준 방식으로 연결하기 위한 프로토콜입니다.
기존에는 시스템마다 API 구조가 달라 연결이 복잡했습니다.
MCP는 이를 표준화하여 다양한 도구를 쉽게 연결할 수 있게 만들어줍니다.
GitHub, Slack, VSCode, Blender 등과 연동할 수 있습니다.
특히 AI Agent가 여러 시스템을 사용하는 데 매우 유용합니다.
Anthropic에서 제안한 개념으로 최근 빠르게 확산되고 있습니다.
AI 도구 생태계를 통합하는 역할을 수행합니다.
미래 AI 플랫폼의 핵심 인터페이스 중 하나로 평가받고 있습니다.

### 9.2 기존 문제

각 시스템마다 API 다름

### 9.3 MCP 해결

```text
LLM
 ↓
MCP Protocol
 ↓
GitHub / DB / Slack / Notion
```

### 9.4 장점

* 표준화
* 도구 연결 단순화
* Agent 확장 쉬움

### 9.5 예시

* GitHub MCP
* Blender MCP
* Roblox MCP
* VSCode MCP

## 10. Workflow Orchestration

### 10.1 역할

Workflow Orchestration은 여러 AI 작업을 자동으로 연결하고 관리하는 기술입니다.
예를 들어 문서 업로드 → OCR → 요약 → Vector DB 저장 → 챗봇 연결 흐름을 자동화할 수 있습니다.
복잡한 AI 파이프라인을 안정적으로 운영할 수 있게 도와줍니다.
특히 Kubernetes와 함께 사용하는 경우가 많습니다.
대표 도구로는 Apache Airflow, Argo Workflows, Kubeflow 등이 있습니다.
MLOps와 DevOps 자동화의 핵심 구성 요소입니다.
AI 서비스 운영 안정성과 재현성을 높일 수 있습니다.
대규모 AI 플랫폼에서는 거의 필수적으로 사용됩니다.

### 10.2 예시

```text
문서 업로드
 ↓
OCR
 ↓
요약
 ↓
벡터화
 ↓
DB 저장
 ↓
챗봇 연결
```

### 10.3 기술

* Apache Airflow
* Argo Workflows
* LangGraph
* Kubeflow

## 11. Multimodal AI

### 11.1 역할

Multimodal AI는 텍스트뿐 아니라 이미지, 음성, 영상 등 다양한 데이터를 함께 이해하는 기술입니다.
예를 들어 이미지를 보고 설명하거나 음성을 텍스트로 변환할 수 있습니다.
YOLO 같은 비전 모델과 LLM을 결합하는 방식도 많이 사용됩니다.
PDF 문서 이해, 영상 분석, 음성 비서 등에 활용됩니다.
최근 GPT 계열 모델들도 멀티모달 기능을 지원하고 있습니다.
사람처럼 여러 감각 정보를 통합 처리하는 방향으로 발전 중입니다.
AI 활용 범위를 크게 확장시킨 핵심 기술입니다.
로봇, 자율주행, 스마트 팩토리 분야에서도 중요성이 계속 커지고 있습니다.

### 11.2 종류

| 입력   | 예시      |
| ------ | --------- |
| 이미지 | 객체 탐지 |
| 음성   | STT       |
| 영상   | 행동 분석 |
| PDF    | 문서 이해 |

### 11.4 예시

YOLO + LLM:

```text
카메라 영상 분석
→ 사람 탐지
→ LLM 설명 생성
```

# 17. 생성형 AI 인프라 구조

생성형 AI 서비스는 대규모 인프라 위에서 운영된다.
사용자는 웹 또는 API를 통해 AI 서비스를 요청한다.
요청은 FastAPI 같은 백엔드 서버로 전달된다.
백엔드는 LLM 서버(vLLM, Ollama 등)에 요청을 전달한다.
LLM 서버는 GPU를 사용하여 추론을 수행한다.
결과는 다시 사용자에게 반환된다.
대규모 환경에서는 Kubernetes 기반으로 운영한다.
Prometheus와 Grafana로 GPU 상태를 모니터링한다.
ArgoCD와 GitOps 기반 자동 배포도 많이 사용된다.
최근에는 GPU 스케줄링과 비용 최적화 기술이 중요해지고 있다.
생성형 AI 인프라는 클라우드·GPU·DevOps·MLOps 기술이 결합된 구조이다.

### 예시 구조

```text id="h3u6v2"
사용자
 ↓
FastAPI
 ↓
vLLM
 ↓
GPU
 ↓
Vector DB
```

### 핵심 정리

* 생성형 AI는 GPU 기반 인프라를 사용한다.
* FastAPI + LLM 서버 구조가 일반적이다.
* Kubernetes 운영이 증가 중이다.
* 모니터링과 자동 배포가 중요하다.
* MLOps 기술과 깊게 연결된다.

# 18. 생성형 AI의 미래

생성형 AI는 앞으로 더욱 강력한 방향으로 발전할 가능성이 높다.
현재는 단순 대화형 AI를 넘어 AI 에이전트 형태로 진화하고 있다.
AI 에이전트는 스스로 계획을 세우고 작업을 수행할 수 있다.
멀티모달 AI는 이미지·음성·영상·텍스트를 동시에 처리한다.
로봇과 결합되면 실제 물리적 작업 수행도 가능해질 수 있다.
개인화 AI는 사용자 습관과 업무 스타일에 맞춰 동작하게 된다.
기업들은 자체 LLM과 프라이빗 AI 구축을 확대하고 있다.
또한 Edge AI 기술이 발전하면서 로컬 장치에서도 AI 실행이 가능해지고 있다.
향후 AI는 운영 자동화와 의사결정 보조 시스템으로 더욱 깊게 통합될 가능성이 높다.
다만 윤리·보안·저작권 문제 해결도 함께 중요해질 것이다.
생성형 AI는 인터넷·모바일 이후 가장 큰 플랫폼 변화가 될 가능성이 높다.

### 예시

```text id="t0j6l4"
미래 AI:
"오늘 일정 정리하고 메일 보내고 회의 자료 준비해줘"

AI Agent:
자동 수행
```

### 핵심 정리

* AI 에이전트 방향으로 발전 중이다.
* 멀티모달 AI가 핵심이다.
* 개인화 AI가 증가할 것이다.
* Edge AI와 로봇 AI가 발전 중이다.
* 윤리와 보안 문제 해결이 중요하다.
