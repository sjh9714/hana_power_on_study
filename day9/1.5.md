## Self-Attention이란?

Self-Attention은 **문장 안의 단어들이 서로 얼마나 중요한 관계인지 계산하는 기술**입니다.

예를 들어 다음 문장을 보겠습니다.

> "철수는 사과를 먹었다."

컴퓨터는 각 단어를 볼 때 다른 단어들과의 관련성을 계산합니다.

| 현재 단어 | 관련 단어 | 중요도(Attention Score) |
| --------- | --------- | ----------------------: |
| 먹었다    | 철수는    | 0.3                     |
| 먹었다    | 사과를    | 0.6                     |
| 먹었다    | 먹었다    | 0.1                     |

여기서 "먹었다"를 이해하려면

* 누가 먹었는지 → 철수
* 무엇을 먹었는지 → 사과

를 알아야 합니다.

그래서 Self-Attention은 "사과를"에 높은 점수(0.6)를 주고, "철수는"에도 점수(0.3)를 줍니다.

---

## 조금 더 쉬운 예제

문장:

> "나는 오늘 피자를 먹었다."

"먹었다"를 이해할 때

| 단어   | 중요도 |
| ------ | -----: |
| 나는   | 0.2    |
| 오늘   | 0.1    |
| 피자를 | 0.6    |
| 먹었다 | 0.1    |

AI는 다음과 같이 생각합니다.

> "먹었다"라는 행동의 대상은 "피자"니까 가장 중요하네!

그래서 "피자를"에 가장 높은 가중치를 부여합니다.

---

## 사람이 문장을 이해하는 방식과 비슷

사람이

> "영희는 학교에서 친구와 공부했다."

를 읽으면

"공부했다"를 이해하기 위해

* 누가? → 영희
* 어디서? → 학교
* 누구와? → 친구

를 동시에 생각합니다.

Self-Attention도 똑같이 각 단어를 볼 때 문장의 다른 단어들을 함께 참고합니다.

---

## 기존 방식(RNN)과 차이

### RNN

```text id="jlwmde"
나는 → 오늘 → 피자를 → 먹었다
```

앞에서 뒤로 순서대로 읽음

---

### Self-Attention

```text id="jlwmfg"
나는 ←→ 오늘
 ↑      ↓
피자를 ←→ 먹었다
```

모든 단어를 한 번에 비교

그래서 학습 속도가 빠르고 긴 문장도 잘 이해할 수 있습니다.

---

## ChatGPT 예제

질문:

> "철수는 사과를 먹고 집에 갔다. 그는 배가 불렀다."

여기서 "그"가 누구인지 알아내려면

| 단어 | 중요도 |
| ---- | -----: |
| 철수 | 0.8    |
| 사과 | 0.1    |
| 집   | 0.1    |

Self-Attention은

> "그" = "철수"

라고 판단합니다.

---

## 한 줄 요약

**Self-Attention은 문장 속 각 단어가 다른 단어들을 얼마나 참고해야 하는지 계산하여 문맥을 이해하는 핵심 기술입니다.**

즉,

> "현재 단어를 이해하기 위해 문장의 어떤 단어를 얼마나 집중해서 볼 것인가?"

를 계산하는 메커니즘이라고 생각하시면 됩니다.

Self-Attention의 핵심 원리를 이해하려면 실제 Transformer 라이브러리보다 **숫자를 직접 계산해보는 것이 가장 좋습니다.**

# 12. Attention Self-Attention 예제

파일명 : self-attention.py

```python
import numpy as np

# ==========================================
# 단어 정의
# ==========================================

words = ["나는", "피자를", "먹었다"]

# ==========================================
# 임의의 벡터(Embedding)
# 실제로는 학습된 값
# ==========================================

embeddings = np.array([
    [1, 0, 1],  # 나는
    [0, 2, 1],  # 피자를
    [1, 1, 2]   # 먹었다
])

print("=== Embedding ===")
for word, emb in zip(words, embeddings):
    print(word, emb)

# ==========================================
# Query = 먹었다
# ==========================================

query = embeddings[2]

print("\nQuery (먹었다):")
print(query)

# ==========================================
# Attention Score 계산
# Dot Product
# ==========================================

scores = []

for emb in embeddings:
    score = np.dot(query, emb)
    scores.append(score)

scores = np.array(scores)

print("\nAttention Scores")
for word, score in zip(words, scores):
    print(word, score)


```

# 13. Softmax 란?

Softmax는 **여러 개의 숫자를 확률(0~1 사이 값)로 변환하는 함수**입니다.

그리고 변환된 값들의 합은 항상 **1(100%)**이 됩니다.

Transformer의 Self-Attention에서는

```text
Attention Score
↓
Softmax
↓
Attention Weight(가중치)
```

과정에서 사용됩니다.

---

## 간단한 예제

세 학생의 시험 점수가 있다고 가정해보겠습니다.

```python
scores = [80, 60, 40]
```

이 점수를 보고

> "누가 가장 잘했는지 확률처럼 표현해줘"

라고 할 때 Softmax를 사용합니다.

---

## Softmax 공식

$$
P(x_i)=\frac{e^{x_i}}{\sum_j e^{x_j}}
$$

여기서

* (x_i) : 현재 값
* (e) : 자연상수(약 2.718)
* 분모 : 전체 값들의 지수합

---

## Python 코드

파일명 : softmax1.py

```python
import numpy as np

scores = np.array([2, 1, 0])

exp_scores = np.exp(scores)

softmax = exp_scores / np.sum(exp_scores)

print(softmax)
```

출력 결과

[0.66524096 0.24472847 0.09003057]

---

## Self-Attention에서의 사용

예를 들어

```text
먹었다
```

가 다른 단어들과 계산한 점수가

```python
scores = [3, 4, 6]
```

라고 가정합니다.

```python

```

```text
[

0.114,
0.844
]
```

## 해석

| 단어   | 점수 | 가중치 |
| ------ | ---: | -----: |
| 나는   | 3    | 4.2%   |
| 피자를 | 4    | 11.4%  |
| 먹었다 | 6    | 84.4%  |

즉

```text
먹었다를 이해할 때

나는      4.2%
피자를   11.4%
먹었다   84.4%
```

만큼 참고하라는 의미입니다.

## 왜 exp(지수)를 사용할까?

만약 단순히 비율만 계산하면

```python

```

결과

```text
[
0.23,
0.31,
0.46
]
```

차이가 크지 않습니다.

---

Softmax는 exp를 사용해서

```text
큰 값은 더 크게
작은 값은 더 작게
```

만들어 줍니다.

```python

=
[20.08,
 54.59,
403.43]
```

이 되므로 중요한 단어가 더욱 강조됩니다.

# 15. Transformer 방식으로 구현

실제 Transformer는

* Query(Q)
* Key(K)
* Value(V)

를 사용합니다.

파일명 : transformer.py

```python
import numpy as np

# =====================================================
# 1. 입력 문장 벡터 (Embedding)
# =====================================================
# 문장:
# "나는 피자를 먹었다"
#
# 실제 GPT에서는 수천 차원의 임베딩 벡터가 사용되지만
# 학습용으로 3차원 벡터를 사용

X = np.array([
    [1, 0, 1],   # 나는
    [0, 2, 1],   # 피자를
    [1, 1, 2]    # 먹었다
])

print("=== 입력 임베딩 ===")
print(X)

# =====================================================
# 2. Query 가중치 행렬 생성
# =====================================================
# 실제 Transformer에서는 학습을 통해 최적화되는 파라미터
#
# Wq : Query 생성용 가중치
# Wk : Key 생성용 가중치
# Wv : Value 생성용 가중치
#
# 현재는 이해를 위해 랜덤값 사용

Wq = np.random.rand(3, 3)
Wk = np.random.rand(3, 3)
Wv = np.random.rand(3, 3)

print("\n=== Wq ===")
print(Wq)

print("\n=== Wk ===")
print(Wk)

print("\n=== Wv ===")
print(Wv)

# =====================================================
# 3. Query 생성
# =====================================================
# 입력 벡터 X와 Wq를 곱하여 Query 생성
#
# Query의 의미:
# "나는 지금 어떤 정보를 찾고 싶은가?"
#
# 예:
# "먹었다"는
# "누가 먹었지?"
# "무엇을 먹었지?"
# 와 같은 정보를 찾으려고 함

Q = X @ Wq

# =====================================================
# 4. Key 생성
# =====================================================
# 입력 벡터 X와 Wk를 곱하여 Key 생성
#
# Key의 의미:
# "내가 가지고 있는 정보는 무엇인가?"
#
# Query가 질문이라면
# Key는 답변 후보

K = X @ Wk

# =====================================================
# 5. Value 생성
# =====================================================
# 입력 벡터 X와 Wv를 곱하여 Value 생성
#
# Value의 의미:
# 실제 전달할 정보
#
# Query와 Key를 비교하여
# 중요도가 높으면 해당 Value를 많이 가져옴

V = X @ Wv

# =====================================================
# 결과 출력
# =====================================================

print("\n============================")
print("Q (Query)")
print("============================")
print(Q)

print("\n============================")
print("K (Key)")
print("============================")
print(K)

print("\n============================")
print("V (Value)")
print("============================")
print(V)
```

# Self-Attention 전체 흐름

```text
입력 단어
   ↓
Embedding
   ↓
Q, K, V 생성
   ↓
Q × Kᵀ
   ↓
Attention Score
   ↓
Softmax
   ↓
Attention Weight
   ↓
Weight × V
   ↓
Context Vector 생성
```

Transformer, BERT, GPT 계열 모델은 모두 이 과정을 수십~수백 층 반복하면서 문맥을 이해합니다.

$$
\mathrm{Attention}(Q,K,V)=\mathrm{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

이 공식 하나가 GPT, BERT, Gemini, Claude 같은 Transformer 모델의 핵심 연산입니다.
