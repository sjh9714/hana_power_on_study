# 파라미터(Parameter)란?

LLM에서 Parameter(파라미터)는:

> “AI가 학습한 지식이 저장된 숫자 값”

입니다.

쉽게 말하면:

```text id="param001"
AI의 기억
AI의 경험
AI의 학습 결과
```

가 숫자로 저장된 것입니다.

---

# 1. 가장 쉬운 한 줄 설명

파라미터는:

> “AI가 정답을 잘 맞추기 위해 조정된 수많은 숫자”

입니다.

---

# 2. 왜 숫자로 저장하는가?

컴퓨터는 결국:

```text id="param002"
숫자 계산만 가능
```

합니다.

따라서 AI는:

* 언어
* 문법
* 의미
* 지식

을 모두 숫자로 저장합니다.

---

# 3. 아주 쉬운 비유 — 요리 레시피

AI를 요리사라고 생각해봅시다.

파라미터는:

```text id="param003"
소금 몇 g
설탕 몇 g
불 세기
조리 시간
```

같은 레시피 값입니다.

---

# 4. 예시

라면 끓이기:

| 파라미터 | 값    |
| -------- | ----- |
| 물 양    | 550ml |
| 스프 양  | 1개   |
| 시간     | 4분   |

이 숫자가 바뀌면 맛이 달라집니다.

AI도 동일합니다.

---

# 5. AI에서의 파라미터

예시:

```text id="param004"
"사과"와 "먹다"는 관련이 높다
```

라는 패턴을 숫자로 저장.

---

# 6. 실제 내부 모습

신경망 내부:

$$
y = wx + b
$$

여기서:

| 기호 | 의미           |
| ---- | -------------- |
| w    | 가중치(weight) |
| b    | 편향(bias)     |

이 값들이 파라미터입니다.

---

# 7. 가장 중요한 것 — Weight(가중치)

Weight는:

```text id="param005"
어떤 정보가 얼마나 중요한가
```

를 나타냅니다.

---

# 8. 쉬운 예시

문장:

```text id="param006"
나는 밥을 먹었다
```

여기서:

```text id="param007"
먹었다 ↔ 밥
```

관계는 강합니다.

따라서 높은 weight.

---

# 9. 반면

```text id="param008"
먹었다 ↔ 자동차
```

는 관련 낮음.

따라서 낮은 weight.

---

# 10. 즉 파라미터란?

AI 내부 연결 강도를 나타내는 숫자.

---

# 11. 실제 느낌

예시:

| 연결          | Weight |
| ------------- | ------ |
| 먹다 ↔ 음식   | 0.95   |
| 먹다 ↔ 자동차 | 0.02   |

---

# 12. 뉴런 비유

신경망은 인간 뇌를 흉내냄.

예시:

```text id="param009"
뉴런 ↔ 뉴런
```

사이 연결 강도가 파라미터.

---

# 13. 파라미터가 많다는 의미

예시:

* 7B 모델 → 70억 개
* 70B 모델 → 700억 개

숫자를 가진다는 뜻.

---

# 14. 왜 이렇게 많은가?

언어는 매우 복잡하기 때문.

예시:

* 문법
* 의미
* 감정
* 상식
* 코드
* 수학

모두 저장 필요.

---

# 15. 실제 비유

파라미터 수 = 뇌세포 연결 수 느낌.

많을수록:

```text id="param010"
더 복잡한 패턴 기억 가능
```

.

---

# 16. 학습 전 상태

처음 AI는:

```text id="param011"
랜덤 숫자
```

상태.

예시:

| Weight |
| -----: |
| 0.001  |
| -0.22  |
| 0.94   |

---

# 17. 학습 과정

문장:

```text id="param012"
나는 밥을 먹었다
```

를 반복 학습.

---

# 18. AI 예측

AI 예측:

| 후보   | 확률 |
| ------ | ---: |
| 먹었다 | 20%  |
| 자동차 | 5%   |

틀림.

---

# 19. 오차 계산

Loss 계산.

예시:

```text id="param013"
정답과 얼마나 차이 나는가
```

---

# 20. 파라미터 수정

오차 줄이도록:

```text id="param014"
weight 값 조금 수정
```

합니다.

---

# 21. 반복 학습

수십억 문장 반복.

결국:

```text id="param015"
먹었다 ↔ 음식
```

관계 강해짐.

---

# 22. 실제로 저장되는 것

AI는 사실:

```text id="param016"
지식을 문장으로 저장하지 않음
```

니다.

대신:

```text id="param017"
수학적 패턴
```

으로 저장.

---

# 23. 매우 중요한 개념

LLM은:

```text id="param018"
위키백과 문장을 그대로 저장
```

하는 것이 아닙니다.

대신:

```text id="param019"
언어 패턴 확률
```

을 학습.

---

# 24. 예시

문장:

```text id="param020"
한국의 수도는 서울이다
```

를 많이 보면:

```text id="param021"
한국 → 수도 → 서울
```

연결 weight 증가.

---

# 25. Attention에도 파라미터 존재

Transformer 내부:

* Q weight
* K weight
* V weight

모두 파라미터.

---

# 26. FFN에도 존재

FFN:

$$
y=xW+b
$$

여기서:

```text id="param022"
W와 b 모두 파라미터
```

.

---

# 27. Embedding도 파라미터

단어 벡터 자체도 학습됨.

예시:

```text id="param023"
고양이 → [0.22, ...]
```

이 숫자들도 파라미터.

---

# 28. 결국 Transformer 전체가 파라미터 덩어리

모든 Layer에:

* Weight
* Bias

존재.

---

# 29. 왜 큰 모델이 똑똑한가?

파라미터 많을수록:

* 더 많은 패턴 저장
* 더 복잡한 관계 학습
* 더 긴 문맥 이해

가능.

---

# 30. 하지만 무조건 큰 게 좋은가?

아닙니다.

너무 크면:

* GPU 메모리 증가
* 속도 감소
* 비용 증가

발생.

---

# 31. 실제 모델 예시

| 모델       | 파라미터 |
| ---------- | -------: |
| GPT-2      | 15억     |
| GPT-3      | 1750억   |
| Llama 3 8B | 80억     |

---

# 32. 메모리 계산 예시

7B 모델:

```text id="param024"
70억 × 2byte(fp16)
≈ 14GB
```

정도 필요.

---

# 33. 추론 시 역할

사용자 질문:

```text id="param025"
한국 수도는?
```

파라미터 기반 계산:

```text id="param026"
한국 ↔ 서울 관계 강함
```

출력:

```text id="param027"
서울
```

---

# 34. Fine-tuning이란?

추가 학습으로:

```text id="param028"
파라미터 재조정
```

하는 것.

---

# 35. LoRA도 파라미터 조정

전체 수정 대신:

```text id="param029"
일부 작은 파라미터만 추가 학습
```

.

---

# 36. 인간 뇌 비유

| 인간      | LLM         |
| -------   | ----------- |
| 경험      | 학습 데이터 |
| 기억      | 파라미터    |
| 사고 연결 | Weight      |

---

# 37. 핵심 직관

파라미터는:

```text id="param030"
AI 내부의 수많은 조절 다이얼
```

같은 것.

---

# 38. 예시 비유

자동차 운전:

| 조절 | 값   |
| ---- | ---- |
| 핸들 | 방향 |
| 엑셀 | 속도 |

AI는 수십억 개 다이얼 존재.

---

# 39. 최종 핵심 정리

파라미터는:

> “AI가 학습한 지식과 패턴을 저장하는 수많은 숫자 값”

입니다.

LLM은 이 거대한 숫자들을 이용해 언어를 이해하고 생성합니다.
