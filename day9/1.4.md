# LLM에서 “벡터(Vector)”란 무엇인가?

LLM에서 벡터는 가장 중요한 핵심 개념 중 하나입니다.

쉽게 말하면:

> “단어의 의미를 숫자 배열로 표현한 것”

입니다.

예를 들어:

```text id="aq6y32"
고양이
```

라는 단어를 컴퓨터는 직접 이해하지 못합니다.

그래서 LLM은 이를:

```text id="e6uqq2"
[0.12, -0.44, 0.91, 0.03, ...]
```

같은 숫자 묶음으로 바꿉니다.

이 숫자 묶음이 바로 벡터입니다.

아래는 Python으로 `"고양이"`를 벡터로 변환하는 예제입니다.

## 방법 1. Hugging Face Sentence Transformer 사용

```python
# 설치
# pip install sentence-transformers

from sentence_transformers import SentenceTransformer

# 한국어 지원 임베딩 모델
model = SentenceTransformer("sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2")

word = "고양이"

# 벡터 변환
vector = model.encode(word)

print("단어:", word)
print("벡터 길이:", len(vector))
print("벡터 일부:", vector[:10])
```

### 방법 2. Transformers 직접 사용

```python
# 설치
# pip install transformers torch

# transformers 라이브러리에서
# 토크나이저(AutoTokenizer)와 모델(AutoModel)을 가져옴
from transformers import AutoTokenizer, AutoModel

# PyTorch 라이브러리
import torch

# 사용할 사전학습 모델 지정
# KLUE(Korean Language Understanding Evaluation)용 BERT 모델
model_name = "klue/bert-base"

# ==========================================================
# 1. 토크나이저 로드
# ==========================================================
# 텍스트를 모델이 이해할 수 있는 토큰(Token)으로 변환
tokenizer = AutoTokenizer.from_pretrained(model_name)

# ==========================================================
# 2. 모델 로드
# ==========================================================
# 사전학습된 BERT 모델을 메모리에 적재
model = AutoModel.from_pretrained(model_name)

# ==========================================================
# 3. 임베딩할 문장 또는 단어
# ==========================================================
text = "고양이"

# ==========================================================
# 4. 토큰화(Tokenization)
# ==========================================================
# return_tensors="pt"
# → 결과를 PyTorch Tensor 형태로 반환
#
# 예)
# "고양이"
#
# inputs =
# {
#   'input_ids': tensor([[2, 5505, 3]]),
#   'token_type_ids': tensor([[0,0,0]]),
#   'attention_mask': tensor([[1,1,1]])
# }
#
inputs = tokenizer(
    text,
    return_tensors="pt"
)

# 토큰화 결과 확인
print("===== Tokenizer 결과 =====")
print(inputs)

# ==========================================================
# 5. 추론(Inference)
# ==========================================================
# no_grad()
# → 학습이 아닌 추론만 수행
# → 메모리 사용량 감소
# → 속도 향상
#
with torch.no_grad():
    outputs = model(**inputs)

# ==========================================================
# 6. 모델 출력 확인
# ==========================================================
#
# outputs.last_hidden_state
#
# shape:
# (배치크기, 토큰수, hidden_size)
#
# 예)
# torch.Size([1, 3, 768])
#
# 1 : 문장 개수
# 3 : [CLS] 고양이 [SEP]
# 768 : BERT 벡터 크기
#
print("\n===== 모델 출력 =====")
print(outputs.last_hidden_state.shape)

# ==========================================================
# 7. [CLS] 벡터 추출
# ==========================================================
#
# BERT 입력 형태
#
# [CLS] 고양이 [SEP]
#
# index
#   0      1      2
#
# [:,0,:]
# ↓
# 모든 배치에서
# 첫 번째 토큰([CLS])의
# 768차원 벡터 추출
#
vector = outputs.last_hidden_state[:, 0, :]

# ==========================================================
# 8. 결과 출력
# ==========================================================
print("\n===== CLS 벡터 =====")

# 벡터 크기
print("벡터 크기:", vector.shape)

# 벡터 일부 출력
print("벡터 일부:")
print(vector[0][:10])

# ==========================================================
# 9. 전체 벡터 확인
# ==========================================================
#
# tensor → numpy 변환
#
embedding = vector[0].numpy()

print("\n===== 전체 임베딩 정보 =====")
print("차원 수:", len(embedding))
print("최소값:", embedding.min())
print("최대값:", embedding.max())
print("평균값:", embedding.mean())

```

### 아주 단순하게 표현하면

LLM 내부에서는 개념적으로 다음과 같이 생각할 수 있습니다.

```python
embedding_table = {
    "고양이": [0.12, -0.44, 0.91, 0.03],
    "강아지": [0.10, -0.41, 0.88, 0.05],
    "자동차": [-0.71, 0.23, -0.11, 0.92]
}

word = "고양이"

vector = embedding_table[word]

print(vector)
```

출력

```text
[0.12, -0.44, 0.91, 0.03]
```

실제 LLM은 이런 벡터를 수백~수천 차원(예: 768, 1024, 4096, 8192 차원 등)으로 관리하며, `"고양이"`와 `"강아지"`는 가까운 위치에, `"자동차"`는 먼 위치에 배치하여 의미를 학습합니다. 이것이 임베딩(Embedding)의 핵심 개념입니다.

---

# 1. 벡터를 왜 사용하는가?

컴퓨터는 결국 숫자 계산만 가능합니다.

즉:

```text id="jj8gbf"
"사과"
```

라는 문자열 자체는 계산이 어렵습니다.

하지만:

```text id="1t0jgr"
[0.3, 0.8, -0.2]
```

처럼 숫자로 표현하면:

* 거리 계산
* 유사도 계산
* 패턴 분석
* 의미 비교

를 할 수 있습니다.

---

# 2. 가장 쉬운 비유 — 좌표

벡터는 좌표라고 생각하면 쉽습니다.

예시:

2차원 공간:

```text id="o2ayqf"
(3, 5)
```

3차원 공간:

```text id="5qevtx"
(1, 7, 2)
```

LLM에서는 이것이 수백~수천 차원으로 확장됩니다.

예시:

```text id="p4r8eu"
(0.12, -0.44, 0.91, 0.22, ... 4096개)
```

---

# 3. 단어를 좌표 공간에 배치

LLM은 단어를 거대한 의미 공간에 배치합니다.

예시:

```text id="h4sv9m"
고양이
강아지
호랑이
```

는 서로 가까운 위치에 놓입니다.

반면:

```text id="f54d1m"
자동차
비행기
컴퓨터
```

는 다른 영역에 위치합니다.

---

# 4. 의미 공간(Semantic Space)

이 공간을:

```text id="yv7z5y"
Semantic Space
```

라고 부릅니다.

즉:

```text id="7p1q8i"
의미 기반 좌표 공간
```

입니다.

---

# 5. 벡터 차원(Dimension)

벡터 길이를 차원이라고 합니다.

예시:

```text id="r98h3z"
[1, 2] → 2차원
[1, 2, 3] → 3차원
```

LLM은 보통:

* 768차원
* 1024차원
* 4096차원
* 8192차원

같은 매우 큰 벡터를 사용합니다.

---

# 6. 왜 차원이 높은가?

단어 의미가 매우 복잡하기 때문입니다.

예를 들어:

```text id="5w4qyf"
은행
```

은:

* 금융기관
* 강가(bank)

라는 여러 의미가 있습니다.

높은 차원은 이런 복잡한 의미를 표현할 수 있게 해줍니다.

---

# 7. 실제 벡터 예시

실제 임베딩 일부 예시:

```text id="h9cw1q"
고양이 →
[0.13, -0.22, 0.91, 0.55, -0.11, ...]
```

```text id="a3jqxg"
강아지 →
[0.15, -0.20, 0.88, 0.50, -0.09, ...]
```

숫자 패턴이 비슷합니다.

그래서:

```text id="wdn39p"
의미도 비슷
```

하다고 판단합니다.

---

# 8. 벡터의 핵심 — 거리 계산

LLM은 벡터 간 거리를 계산합니다.

대표적으로:

* Euclidean Distance
* Cosine Similarity

를 사용합니다.

---

# 9. Cosine Similarity (코사인 유사도)

가장 많이 사용됩니다.

두 벡터 방향이 얼마나 비슷한지 계산합니다.

$$
\cos(\theta)=\frac{A\cdot B}{||A||||B||}
$$

---

# 10. 쉬운 의미

| 값  | 의미        |
| :-: | ----------- |
| 1   | 완전히 비슷 |
| 0   | 관련 없음   |
| -1  | 반대        |

---

# 11. 예시

파일명 : vector3.py

```python
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

# 1. 한국어/영어 모두 잘 지원하는 멀티링구얼 모델 로드
model = SentenceTransformer('jhgan/ko-sroberta-multitask')

# 2. 문맥(Context)이 포함된 문장으로 정의
sentences = [
    "동물학적으로 분류했을 때 고양잇과에 속하는 고양이",  # 인덱스 0
    "동물학적으로 분류했을 때 고양잇과에 속하는 호랑이",      # 인덱스 1
    "공학적으로 분류했을 때 기계에 속하는 자동차"     # 인덱스 2
]

# 3. 문장을 임베딩 벡터로 변환
embeddings = model.encode(sentences)

cat_vector = [embeddings[0]]
dog_vector = [embeddings[1]]
car_vector = [embeddings[2]]

# 4. 코사인 유사도 계산
cat_dog_sim = cosine_similarity(cat_vector, dog_vector)[0][0]
cat_car_sim = cosine_similarity(cat_vector, car_vector)[0][0]

# 5. 결과 출력
print("=== 문맥을 반영한 유사도 결과 ===")
print(f"고양이 ↔ 호랑이 유사도: {cat_dog_sim:.4f}")
print(f"고양이 ↔ 자동차 유사도: {cat_car_sim:.4f}")
```


```text id="x0gx0r"
고양이 ↔ 호랑이 유사도: 0.6883
고양이 ↔ 자동차 유사도: 0.1281
```

즉:

```text id="z1fclu"
고양이와 호랑이는 매우 유사
```

하다고 판단합니다.

# 14. 벡터의 각 숫자는 무엇인가?

매우 중요한 질문입니다.

예시:

```text id="d1wdwl"
[0.13, -0.22, 0.91, ...]
```

여기서 각 숫자는:

```text id="v5g7n4"
특정 의미 특징(feature)
```

을 나타냅니다.

다만 인간이 직접 해석하기는 어렵습니다.

---

# 15. 예시 비유

가상의 의미:

| 차원 | 의미(가정) |

| 3번  | 움직임     |
| 4번  | 생명체     |

실제 모델은 이런 특징을 자동 학습합니다.

# 16. 임베딩(Embedding)

단어를 벡터로 바꾸는 과정을:

```text id="xf2xth"
Embedding
```

이라고 합니다.

---

# 17. Embedding Layer

LLM 내부 첫 단계 중 하나입니다.

```text id="w4n9a9"
Token ID
↓
Embedding Layer
↓
Vector
```


됩니다.

# 21. 예시

문장에서:

```text id="r6k1bm"
고양이가 생선을 먹었다
```

```text id="5r4krv"
강아지가 사료를 먹었다
```

패턴이 비슷하므로:

```text id="swc5cc"
고양이 ↔ 강아지
```

벡터도 가까워집니다.

```python
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

# 1. 한국어/영어 모두 잘 지원하는 멀티링구얼 모델 로드
model = SentenceTransformer('jhgan/ko-sroberta-multitask')

# 2. 문맥(Context)이 포함된 문장으로 정의
sentences = [
    "동물학적으로 분류했을 때 고양잇과에 속하는 고양이",  # 인덱스 0
    "동물학적으로 분류했을 때 고양잇과에 속하는 호랑이",      # 인덱스 1
]

# 3. 문장을 임베딩 벡터로 변환
embeddings = model.encode(sentences)

cat_vector = [embeddings[0]]
dog_vector = [embeddings[1]]
car_vector = [embeddings[2]]

# 4. 코사인 유사도 계산
cat_dog_sim = cosine_similarity(cat_vector, dog_vector)[0][0]
cat_car_sim = cosine_similarity(cat_vector, car_vector)[0][0]

# 5. 결과 출력
print("=== 문맥을 반영한 유사도 결과 ===")
print(f"고양이 ↔ 호랑이 유사도: {cat_dog_sim:.4f}")
print(f"고양이 ↔ 자동차 유사도: {cat_car_sim:.4f}")

```

실행결과

```bash
=== 단어 유사도 결과 ===
고양이 ↔ 강아지 유사도: 0.1260
=== 문맥을 반영한 유사도 결과 ===
고양이 ↔ 강아지 유사도: 0.4136
```
