# Positional Encoding 이란?

Positional Encoding은:

> “단어가 문장 안에서 몇 번째인지 알려주는 번호표”

입니다.

---

# 1. 왜 번호표가 필요한가?

Transformer는 문장을:

```text id="easy002"
동시에 한꺼번에 봄
```

니다.

즉:

```text id="easy003"
나는 / 학교에 / 갔다
```

를 순서대로 읽는 것이 아니라:

```text id="easy004"
모든 단어를 동시에 처리
```

합니다.

---

# 2. 문제 발생

동시에 보면:

```text id="easy005"
누가 먼저인지 모름
```

니다.

예시:

```text id="easy006"
나는 학교에 갔다
```

와

```text id="easy007"
학교에 나는 갔다
```

를 비교해봅시다.

---

# 3. 단어만 보면 동일

두 문장은 단어 종류가 같습니다.

```text id="easy008"
나는
학교에
갔다
```

만 존재.

Transformer가 위치를 모르면:

```text id="easy009"
둘이 거의 같은 문장처럼 보임
```

니다.

---

# 4. 그래서 위치 번호를 붙임

예시:

| 단어   | 위치 |
| ------ | ---: |
| 나는   | 1    |
| 학교에 | 2    |
| 갔다   | 3    |

즉:

```text id="easy010"
단어 + 위치번호
```

를 같이 전달.

---

# 5. 학교 자리 비유

학생들이 있다고 가정.

| 학생 | 자리 |
| ---- | ---: |
| 철수 | 1번  |
| 영희 | 2번  |
| 민수 | 3번  |

만약 자리 정보 없으면:

```text id="easy011"
누가 어디 앉았는지 모름
```

니다.

Transformer도 동일합니다.

---

# 6. 단어 의미만으로는 부족

예시:

```text id="easy012"
사과
```

라는 단어만 보면:

* 주어인지
* 목적어인지
* 어디 위치인지

모릅니다.

---

# 7. 실제 예시

문장:

```text id="easy013"
철수가 사과를 먹었다
```

위치 포함:

| 위치 | 단어   |
| ---- | ------ |
| 1    | 철수가 |
| 2    | 사과를 |
| 3    | 먹었다 |

---

# 8. Position이 없다면?

Transformer는:

```text id="easy014"
먹었다
```

가:

* 누가 먹었는지
* 무엇을 먹었는지

파악 어려움.

---

# 9. 사람은 순서를 자동 이해

사람은:

```text id="easy015"
철수가 사과를 먹었다
```

를 읽으면:

* 철수 = 주체
* 사과 = 음식

자동 이해.

---

# 10. Transformer는 숫자 계산기

하지만 Transformer는:

```text id="easy016"
숫자 계산만 수행
```

합니다.

즉:

```text id="easy017"
순서를 따로 알려줘야 함
```

니다.

---

# 11. 실제 내부 느낌

원래 단어 벡터:

| 단어 | 벡터       |
| ---- | ---------- |
| 철수 | [1.2, 0.3] |
| 사과 | [0.7, 1.1] |

---

# 12. 위치 벡터 추가

위치 벡터:

| 위치 | 벡터       |
| ---- | ---------- |
| 1번  | [0.1, 0.1] |
| 2번  | [0.2, 0.2] |

---

# 14. 실제 합산

철수:

```text id="easy018"
[1.2, 0.3]
+
[0.1, 0.1]
=
[1.3, 0.4]
```

사과:

```text id="easy019"
[0.7, 1.1]
+
[0.2, 0.2]
=
[0.9, 1.3]
```

---

# 14. 결과

이제 Transformer는:

```text id="easy020"
단어 의미 + 위치
```

를 동시에 알 수 있음.

---

# 15. 도로 주소 비유

단어만 있으면:

```text id="easy021"
"김철수"
```

라는 이름만 있는 상태.

하지만 주소 추가:

```text id="easy022"
서울 3번지 김철수
```

면 위치 구분 가능.

Positional Encoding이 주소 역할.

---

# 16. 책 문장 비유

예시:

```text id="easy023"
"개가 사람을 물었다"
```

와

```text id="easy024"
"사람이 개를 물었다"
```

는 완전히 다름.

하지만 단어만 보면:

```text id="easy025"
개 / 사람 / 물었다
```

동일.

---

# 17. 위치가 의미를 결정

즉:

```text id="easy026"
순서 자체가 의미
```

입니다.

---

# 18. 왜 단순 숫자 안 쓰는가?

질문:

```text id="easy027"
그냥 위치 1,2,3 넣으면 안되나?
```

가능하지만 문제가 있음.

---

# 19. 단순 숫자의 문제

예시:

| 위치 | 값   |
| ---- | ---- |
| 1    | 1    |
| 2    | 2    |
| 1000 | 1000 |

값이 너무 커질 수 있음.

또:

```text id="easy028"
거리 패턴 학습 어려움
```

니다.

---

# 20. 그래서 sin/cos 사용

Transformer 논문은:

```text id="easy029"
파도(wave) 형태
```

를 사용.

---

# 21. 파동 느낌 이해

sin 그래프 느낌:

```text id="easy030"
~ ~ ~ ~ ~
```

위치가 증가하면 값이 부드럽게 변함.

---

# 22. 왜 파동이 좋은가?

가까운 위치는:

```text id="easy031"
비슷한 패턴
```

을 가짐.

예시:

| 위치 | 값   |
| ---- | ---- |
| 1    | 0.84 |
| 2    | 0.90 |

---

# 23. Transformer가 배우는 것

Transformer는:

```text id="easy032"
"2번 단어와 3번 단어는 가까움"
```

같은 관계를 이해 가능.

---

# 24. 음악 박자 비유

음악도:

```text id="easy033"
위치(박자)
```

가 중요.

같은 음이라도 위치 다르면 다른 음악.

문장도 동일.

---

# 25. 실제 Attention과 연결

Attention은:

```text id="easy034"
어떤 단어를 참고할지 계산
```

합니다.

Positional Encoding 덕분에:

```text id="easy035"
가까운 단어
앞 단어
뒤 단어
```

도 고려 가능.

---

# 26. 예시

문장:

```text id="easy036"
나는 오늘 밥을 먹었다
```

"먹었다"는:

* 밥을 강하게 참고
* 오늘은 시간 정보
* 나는 주체

참고.

위치가 중요.

---

# 27. 실제 내부 느낌

Transformer 입력:

| 단어 | 의미 벡터 | 위치 벡터 |
| ---- | --------- | --------- |
| 나는 | 의미      | 위치1     |
| 오늘 | 의미      | 위치2     |
| 밥을 | 의미      | 위치3     |

합쳐서 처리.

---

# 29. 핵심 직관

Transformer는 원래:

```text id="easy037"
단어 순서를 모르는 AI
```

입니다.

Positional Encoding은:

```text id="easy038"
문장 번호표 시스템
```

입니다.
