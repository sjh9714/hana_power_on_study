금융 데이터에서 **LOCATION(위치) 구조**는 단순히 “어디서 결제했는가”를 넘어서, **거래의 물리적 현실성·이상성·이동 패턴·도용 여부**를 판단하는 핵심 축입니다.

특히 카드 결제/금융 거래 데이터에서는 위치 정보가 **사기 탐지(Fraud Detection)의 핵심 변수**로 쓰입니다.

---

# 1. LOCATION 구조의 기본 의미

금융 데이터의 위치 정보는 보통 2개의 좌표쌍으로 구성됩니다:

## (1) 고객 위치 (Customer Location)

* `lat` → 고객 위도
* `long` → 고객 경도

👉 의미:

* 카드 소유자의 “등록 주소” 또는 “현재 위치”

---

## (2) 가맹점 위치 (Merchant Location)

* `merch_lat` → 가맹점 위도
* `merch_long` → 가맹점 경도

👉 의미:

* 실제 결제가 발생한 장소

---

## (3) 행정 단위 위치 정보

* `city`
* `state`
* `zip`

👉 의미:

* 좌표를 사람이 이해 가능한 지역으로 변환한 값

---

# 2. LOCATION 구조가 중요한 이유

위치 데이터의 핵심 목적은 단순 지도 표시가 아니라:

> “이 거래가 물리적으로 가능한가?”를 판단하는 것

---

## 핵심 질문 3가지

### ① 고객이 실제로 그 장소에 있었는가?

* 서울에 있는 고객이
* 10분 뒤 뉴욕에서 결제?

👉 현실적으로 불가능 → 사기 가능성 ↑

---

### ② 이동 속도가 가능한가?

* 1시간 전에 부산
* 현재 서울에서 결제

👉 이동 거리 / 시간 = 속도 계산

---

### ③ 비정상 패턴인가?

* 평소 서울만 사용
* 갑자기 해외 결제

👉 행동 패턴 변화 탐지

---

# 3. 핵심 분석: 거리(Distance) 계산

LOCATION 구조에서 가장 중요한 파생 변수는 **거리(distance)** 입니다.

---

## 3-1. 고객 ↔ 가맹점 거리

고객 위치와 가맹점 위치 사이 거리:

$$
d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}
$$

👉 의미:

* 고객이 실제로 이동해서 결제했는지 추정

---

## 금융에서의 해석

| 거리       | 의미                              |
| ---------- | --------------------------------- |
| 0 ~ 1km    | 정상 (근거리 결제)                |
| 1 ~ 50km   | 가능 (이동 가능)                  |
| 50 ~ 500km | 의심                              |
| 1000km+    | 매우 위험 (사기 가능성 매우 높음) |

---

# 4. LOCATION 기반 핵심 Feature Engineering

LOCATION 데이터는 그대로 쓰지 않고 반드시 “파생 변수”로 변환합니다.

---

## 4-1. 거리 기반 Feature

### (1) 고객 ↔ 가맹점 거리

```python
distance_km
```

👉 가장 중요한 fraud feature

---

### (2) 이동 속도 (Velocity)

```text
speed = distance / time_diff
```

👉 의미:

* 인간이 이동 가능한 속도인지 판단

예:

* 500km / 10분 = 불가능 → 사기

---

### (3) 위치 변화 여부

```text
is_location_changed = 1 or 0
```

👉 의미:

* 평소 위치와 다른 지역 결제 여부

---

# 5. LOCATION 기반 이상 탐지 시나리오

## 시나리오 1: 카드 도용

* 고객 등록 위치: 서울
* 결제 위치: 미국 뉴욕
* 시간 차이: 5분

👉 결과:

* 물리적으로 불가능
* Fraud 확률 매우 높음

---

## 시나리오 2: 정상 여행

* 고객 위치: 서울
* 10시간 후 일본 도쿄 결제

👉 결과:

* 가능 (여행 패턴)

👉 핵심:
→ “시간 + 이동 패턴” 같이 봐야 함

---

## 시나리오 3: 반복 이상 패턴

* 새벽 2시
* 서로 다른 국가에서 연속 결제

👉 결과:

* 자동화 공격 가능성

---

# 6. LOCATION 구조의 실제 분석 로직

금융 시스템에서는 다음과 같은 규칙을 사용합니다:

---

## Rule 1: 거리 Threshold

```text
if distance_km > 1000:
    risk += HIGH
```

---

## Rule 2: 속도 제한

```text
if speed > 900 km/h:
    fraud_score += HIGH
```

👉 인간 이동 속도 초과 = 의심

---

## Rule 3: 지역 변화 패턴

```text
if country_changed == True and time_diff < 1 hour:
    fraud_score += HIGH
```

---

# 7. LOCATION 데이터의 핵심 가치

LOCATION 구조는 단순 위치 정보가 아니라:

## 👉 “물리적 현실 검증 시스템”

입니다.

---

## 금융에서 LOCATION이 하는 역할

### 1. 신원 검증

* 실제 사용자 위치인지 확인

### 2. 이상 거래 탐지

* 불가능한 이동 탐지

### 3. 행동 패턴 분석

* 평소 생활권 분석

### 4. 자동 Fraud Scoring

* 머신러닝 feature 핵심

---

# 8. LOCATION 데이터의 특징 (중요)

## (1) 단독으로는 의미 없음

* 위치 하나만 보면 부족
* 시간/금액/사용패턴과 결합 필요

---

## (2) 항상 “관계형 데이터”이다

LOCATION은 독립 데이터가 아니라:

* 고객 ↔ 위치
* 거래 ↔ 위치
* 시간 ↔ 위치

의 관계 구조

---

## (3) 가장 강력한 이상 탐지 feature 중 하나

실제 금융 fraud 모델에서:

> LOCATION feature는 상위 1~3위 중요도

---

# 9. 핵심 정리

금융 데이터에서 LOCATION 구조란:

> “고객과 거래 장소의 공간적 관계를 기반으로, 실제 이동 가능성과 거래의 정상 여부를 판단하는 핵심 분석 구조”
