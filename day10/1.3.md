# 금융 데이터 분석 프롬프트 실습은 어떻게 해야 하는가?

금융 데이터 분석에서 프롬프트 실습의 핵심은 단순히:

> “데이터를 분석해줘”

가 아니라,

> “AI가 금융 데이터를 정확하게 해석하고 분석하도록 구조화된 분석 지시를 설계하는 것”

입니다.

특히 금융 분야는:

* 이상 탐지(Fraud Detection)
* 고객 소비 패턴 분석
* 리스크 분석
* 시계열 분석
* 투자 분석
* 신용 평가
* 거래 이상징후 탐지

등 다양한 분석 유형이 존재하기 때문에, 프롬프트 구조가 매우 중요합니다.

---

# 금융 데이터 분석 프롬프트 실습의 핵심 목표

실습 목표는 보통 다음과 같습니다.

| 목표           | 설명               |
| -------------- | ------------------ |
| 데이터 이해    | 컬럼/패턴 파악     |
| 통계 분석      | 평균/분산/상관관계 |
| 이상 거래 탐지 | Fraud Detection    |
| 고객 세분화    | 소비 패턴 분석     |
| 시각화         | 그래프/히트맵      |
| 모델 생성      | ML/DL 적용         |
| 결과 해석      | 금융 관점 분석     |
| 자동 리포트    | 보고서 생성        |

---

```text id="7ovru4"
3. 이상치 가능 컬럼
4. 타겟 컬럼 추정
5. 데이터 품질 문제
를 분석해줘.
```

# 3단계: 이상 거래 탐지 실습

금융 AI에서 가장 핵심적인 실습입니다.

---

# 추천 프롬프트

```text id="cb6is7"
너는 금융 AI 엔지니어이다.

credit_card_transactions.csv 데이터를 기반으로
신용카드 이상 거래 탐지를 수행해줘.

조건:
- 정상/사기 거래 비율 분석
- 클래스 불균형 해결
- XGBoost 사용
- Precision/Recall/F1 출력
- Threshold 0.3~0.9 비교
- 중요 Feature 시각화
- 실무 관점 해석 포함
```

---

# 여기서 핵심

금융 데이터는:

* 클래스 불균형
* 고객별 편차
* 시간 패턴
* 지역 패턴

이 매우 중요합니다.

즉:

> 단순 정확도(Accuracy)만 보면 안 됨

---

# 금융 데이터에서는 Precision/Recall이 중요

특히 Fraud Detection에서는:

```text id="4r5edx"
Precision = 실제 사기 비율 정확성
Recall = 실제 사기를 얼마나 놓치지 않는가
```

가 매우 중요합니다.

---

---

# 추천 프롬프트

```text id="t5lz3w"

```

# 여기서 중요한 금융 개념

금융 데이터는:

> 전체 평균보다 개인 패턴이 중요

합니다.

예:

* 어떤 고객은 새벽 2시 거래가 정상
* 어떤 고객은 새벽 거래 자체가 이상

즉:

> 개인 기준 이상 탐지(Persona-based Detection)

가 중요합니다.

* 불균형 데이터
* 이상치 중심
* 시간 패턴 중요
* 고객 개인화 중요
* 설명 가능성(XAI) 중요

합니다.

따라서 프롬프트에도 반드시:

* Precision/Recall
* Explainability
* Threshold
* 고객 패턴
* 시계열 특성

을 포함하는 것이 좋습니다.

# 최종 핵심 정리

금융 데이터 분석 프롬프트 실습의 핵심은:

> “AI가 금융 데이터를 금융 관점으로 해석하도록 구조화하는 것”

입니다.

즉 단순 데이터 분석이 아니라:

* 이상 거래 관점
* 리스크 관점
* 고객 패턴 관점
* 시계열 관점
* 운영 자동화 관점

으로 프롬프트를 설계하는 것이 가장 중요합니다.
